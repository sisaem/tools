브라우저 기반 PDF 처리와 보안 안내

온라인 PDF 도구를 사용할 때 가장 많이 걱정하는 부분은 파일이 서버로 업로드되는지, 다른 사람이 내용을 볼 수 있는지입니다. Sisaemdal Tools는 가능한 작업을 사용자의 브라우저에서 처리하도록 구성되어 있습니다.

이 안내는 일반적인 PDF 작업을 돕기 위한 설명입니다. 중요한 문서는 원본을 별도로 보관하고, 결과 파일을 직접 확인한 뒤 사용하세요.

브라우저 처리란 무엇인가

브라우저 처리란 사용자가 선택한 파일을 웹페이지 안의 자바스크립트와 라이브러리가 사용자의 기기에서 읽고 처리하는 방식입니다. 별도의 서버 업로드 기능을 두지 않으면 운영자가 파일 원본을 직접 열람하거나 저장하지 않습니다. 다만 외부 라이브러리 파일을 불러오기 위한 네트워크 요청은 발생할 수 있습니다.

F12 개발자 도구에 대해

방문자는 F12 개발자 도구로 자기 브라우저 안의 HTML, CSS, JavaScript, 네트워크 요청을 볼 수 있습니다. 이것은 웹의 기본 특성입니다. 사용자가 자기 화면에서 버튼을 숨기거나 문구를 바꾸는 것은 가능하지만, 그것이 서버의 원본 파일이나 다른 방문자의 화면을 바꾸는 것은 아닙니다.

민감한 파일 사용 시 주의

공용 PC, 원격 제어 중인 PC, 악성 확장 프로그램이 설치된 브라우저에서는 민감한 문서를 처리하지 않는 것이 좋습니다. PDF에 주민등록번호, 계좌번호, 계약 정보, 내부 문서가 포함되어 있다면 작업 후 다운로드 폴더와 브라우저 기록도 함께 확인하세요.

운영자가 권장하는 사용 방식

중요한 문서는 원본을 보관한 상태에서 사본으로 작업하고, 결과 파일을 열어 내용과 페이지 수를 확인하세요. 공유 전에는 불필요한 페이지와 개인정보가 남아 있지 않은지 다시 확인하는 것이 안전합니다.
